03 加电自检与计算机的启动过程

TanG 2024-11-26 15:52:26
Categories： Tags：

POST（Power on self testing）

计算机在启动时，首先不是操作系统运行，而是BIOS，BIOS固定在主板，与操作系统无关。

在操作系统运行之前，就是BIOS管理计算机，运行完毕，让操作系统接管计算机

1. POST上电自检

计算机接通电源后，系统首先由POST（Power On Self Test，上电自检）程序来对内部各个设备进行检查。通常完整的POST自检包括对CPU、640KB基本内存、1MB以上的扩展内存、ROM、主板、CMOS存储器、串并口、显卡、软硬盘子系统及键盘进行测试，一旦在自检中发现问题，系统将给出提示信息或鸣笛警告。

2. BIOS系统启动自举程序

系统完成POST自检后，ROM BIOS就首先按照系统CMOS设置中保存的启动顺序搜索软硬盘驱动器及CD-ROM、网络服务器等有效的启动驱动器，读入操作系统引导记录，然后将系统控制权交给引导记录，并由引导记录来完成系统的顺序启动。

BIOS 存储区域的物理分布主要包括以下几部分：

  • 主板上的 BIOS 芯片：
    • BIOS 通常存储在主板上的一个专用闪存芯片中，这个芯片被连接到主板的一个特定插槽上，通常会标识为 BIOS、UEFI 或 FLASH。
    • 现代计算机的 BIOS 大多存储在一个大小为 4MB 至 32MB 的闪存芯片中，这个芯片通常位于主板的中心位置，靠近 CPU 和内存插槽。
  • BIOS 存储分布：
    • BIOS 存储区域通常会分为多个部分：
      1. 启动代码（Boot Code）：负责计算机开机时初始化硬件并加载操作系统引导程序（如引导扇区或引导加载程序）。
      2. 硬件初始化（Hardware Initialization）：包含对硬件组件的初始化程序，确保 CPU、内存、硬盘、显示适配器等设备能够正常工作。
      3. 系统设置（System Settings）：包括一些基本配置选项（如时钟、引导顺序、硬盘设置等），通常可以通过 BIOS 设置界面（如 “Setup” 或 “UEFI Setup”）进行修改。
      4. 驱动程序（Drivers）：一些最基本的硬件驱动程序，提供对计算机硬件的初步支持。

BIOS的启动过程：

当计算机启动时，CPU 会首先执行 POST（Power-On Self Test），然后加载和执行存储在 BIOS 中的启动程序。启动过程大致如下：

  1. 加电自检（POST）：开机后，CPU 会开始执行 BIOS 中的自检程序，检查系统硬件（如内存、CPU、显卡等）是否正常。
  2. 硬件初始化：BIOS 会初始化计算机的硬件，设置硬件参数，配置外围设备。
  3. 加载引导程序：完成硬件初始化后，BIOS 会查找并加载引导程序，通常是操作系统的启动引导程序。BIOS 会根据用户设置的启动顺序，查找硬盘、U盘、光盘等设备中的引导扇区。
  4. 启动操作系统：一旦找到有效的引导程序，BIOS 会将控制权交给操作系统的引导程序（例如，GRUB、Windows Boot Manager等），操作系统开始加载。
